# WCAG 2.2 AA 準拠チェックリスト

## 1. 知覚可能 (Perceivable)

### 1.1 テキスト代替 (Level A)

- [ ] すべての `<img>` に意味のある `alt` 属性がある
- [ ] 装飾画像は `alt=""` で空にされている
- [ ] `<svg>` に `aria-label` または `<title>` がある
- [ ] アイコンボタンに `aria-label` がある
- [ ] CAPTCHAにテキスト代替がある

### 1.2 時間依存メディア (Level A/AA)

- [ ] 動画に字幕がある (Level A)
- [ ] 動画に音声解説がある (Level AA)
- [ ] 自動再生される動画は停止可能

### 1.3 適応可能 (Level A)

- [ ] 情報が視覚的な配置だけに依存していない
- [ ] 見出し（h1-h6）が論理的な階層になっている
- [ ] フォーム要素に `<label>` が関連付けられている
- [ ] テーブルに `<th>` と `scope` 属性がある
- [ ] 読み上げ順序が視覚的な順序と一致

### 1.4 判別可能 (Level AA)

- [ ] **テキストと背景の色コントラスト比が4.5:1以上**（通常テキスト）
- [ ] **大きなテキストのコントラスト比が3:1以上**（18pt以上 or 14pt太字）
- [ ] テキストが200%まで拡大可能
- [ ] 画像内のテキストを避けている（ロゴ除く）
- [ ] コンテンツが横スクロールなしで320px幅で表示可能（1.4.10 Reflow）
- [ ] フォーカスインジケータのコントラスト比が3:1以上（1.4.11）
- [ ] テキスト間隔の調整が可能（1.4.12）

---

## 2. 操作可能 (Operable)

### 2.1 キーボード操作 (Level A)

- [ ] **すべての機能がキーボードのみで操作可能**
- [ ] キーボードフォーカスがトラップされない（モーダル内は例外）
- [ ] ショートカットキーがある場合、無効化/再割り当て可能

### 2.2 十分な時間 (Level A)

- [ ] 時間制限がある場合、延長/解除/警告が可能
- [ ] 自動更新コンテンツは一時停止/停止/非表示が可能

### 2.4 ナビゲーション可能 (Level A/AA)

- [ ] 「本文へスキップ」リンクがある (Level A)
- [ ] ページタイトルが適切 (Level A)
- [ ] **フォーカス順序が論理的** (Level A)
- [ ] リンクの目的がテキストから理解可能 (Level A)
- [ ] 複数の方法でページに到達可能（ナビゲーション + サイトマップ/検索） (Level AA)
- [ ] 見出しとラベルが内容を説明している (Level AA)
- [ ] **フォーカスが視覚的に分かる** (Level AA)

### 2.5 入力モダリティ (Level A/AA)

- [ ] タッチターゲットが24x24px以上（2.5.8, Level AA）
- [ ] ドラッグ操作に代替手段がある（2.5.7, Level AA）
- [ ] ジェスチャー操作に単一ポインタの代替がある

---

## 3. 理解可能 (Understandable)

### 3.1 読み取り可能 (Level A)

- [ ] ページの言語が `<html lang="ja">` で指定されている
- [ ] 部分的に異なる言語は `lang` 属性で指定

### 3.2 予測可能 (Level A/AA)

- [ ] フォーカスを受けただけでコンテキストが変わらない (Level A)
- [ ] 入力しただけで（送信前に）コンテキストが変わらない (Level A)
- [ ] ナビゲーションが一貫している (Level AA)
- [ ] 同じ機能は同じように識別される (Level AA)

### 3.3 入力支援 (Level A/AA)

- [ ] **エラーが自動検出されたら、テキストでユーザーに通知** (Level A)
- [ ] フォーム要素にラベルまたは説明がある (Level A)
- [ ] **エラー修正の提案がある** (Level AA)
- [ ] 重要な操作は確認/取り消し/確認ステップがある (Level AA)

---

## 4. 堅牢 (Robust)

### 4.1 互換性 (Level A/AA)

- [ ] HTMLの構文が正しい（開始/終了タグの対応、id重複なし）
- [ ] **すべてのUI要素にname, role, valueが正しく設定**
- [ ] ステータスメッセージがプログラム的に判定可能（4.1.3, Level AA）
