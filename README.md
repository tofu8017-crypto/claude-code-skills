# Claude Code Skills Collection

Claude Code で使えるスキル（＝専門知識パック）のコレクションです。

---

## これは何？

**Claude Code**（AI アシスタント）に「専門知識」を追加するファイル集です。

たとえば、あなたが Claude Code に「APIを設計して」と頼んだとき、このスキルがあると **RESTful設計のベストプラクティスに従った、プロ品質のAPI仕様書** を自動生成してくれます。スキルがないと、一般的な回答しか返ってきません。

**たとえるなら**: Claude Code が「汎用的な頭のいい人」だとしたら、スキルは「その分野の専門マニュアル」です。マニュアルを渡すことで、専門家のように振る舞えるようになります。

---

## 収録スキル一覧

| スキル名 | 何をしてくれるか | こう言えば動く |
|---------|----------------|--------------|
| **api-design-generator** | REST API の設計図（OpenAPI仕様書）を作る | 「APIを設計して」「エンドポイントを設計したい」 |
| **performance-optimizer** | サイトの遅さを診断して、具体的な高速化策を提示 | 「表示が遅い」「パフォーマンスを改善したい」 |
| **a11y-audit** | 障害のある方も使えるサイトか監査し、修正する | 「アクセシビリティをチェックして」「a11y監査して」 |
| **dependency-audit** | 使っているライブラリの安全性・ライセンス・不要物を検査 | 「依存関係を監査して」「使ってないパッケージを見つけて」 |

## 各スキルのかんたん解説

### api-design-generator（API設計ジェネレーター）

**API** = アプリ同士がデータをやりとりする窓口のこと。

たとえば「ユーザー一覧を取得する」「新しい商品を登録する」といった操作をどういうURL・形式で行うかの **設計図** を作ってくれます。設計図は「OpenAPI」という世界標準のフォーマットで出力されるので、そのまま開発チームに渡せます。

### performance-optimizer（パフォーマンス最適化）

**パフォーマンス** = サイトの表示速度や操作のサクサク感のこと。

「ページが開くのに5秒かかる」「スクロールがカクカクする」といった問題の **原因を特定** し、「この画像を最適化すれば2秒速くなる」「このコードをこう変えれば90%速くなる」のように **数字付きの改善策** を出してくれます。

### a11y-audit（アクセシビリティ監査）

**アクセシビリティ** = 障害のある方や、さまざまな環境の方でもサイトを使えるようにすること。

目が見えない方はスクリーンリーダー（音声読み上げ）でサイトを閲覧しますが、画像に説明文がないと「何の画像か」わかりません。キーボードだけで操作する方もいます。このスキルは、そうした **「使えない箇所」を見つけて、具体的な修正コードを出してくれます**。

日本でも法律（障害者差別解消法）で対応が求められるケースが増えており、ビジネス上も重要です。

### dependency-audit（依存関係の監査）

**依存関係** = あなたのプロジェクトが使っている外部ライブラリ（他の人が作った部品）のこと。

Webアプリは多くの場合、数百個のライブラリを組み合わせて作られています。このスキルは：
- **セキュリティの穴**（脆弱性）がないか検査
- **ライセンス**（利用条件）が商用利用OKか確認
- **使っていないのに入っているライブラリ** を見つけて削除提案
- **古くなったライブラリ** の安全な更新計画を立案

してくれます。

## インストール方法

### 方法1: フォルダごとコピー（かんたん）

1. このリポジトリをダウンロード（緑の「Code」ボタン → 「Download ZIP」）
2. ZIPを解凍
3. `skills/` フォルダの中にある4つのフォルダを、以下の場所にコピー:

```
Mac/Linux:  ~/.claude/skills/
Windows:    %USERPROFILE%\.claude\skills\
```

### 方法2: コマンドでインストール

ターミナル（コマンドプロンプト）で以下を実行:

```bash
# リポジトリをクローン
git clone https://github.com/tofu8017-crypto/claude-code-skills.git

# スキルフォルダにコピー
cp -r claude-code-skills/skills/* ~/.claude/skills/
```

### インストール確認

Claude Code を起動して、以下のように話しかけてみてください:

- 「APIを設計して」→ api-design-generator が起動すればOK
- 「アクセシビリティをチェックして」→ a11y-audit が起動すればOK

## フォルダ構成

```
skills/
├── api-design-generator/        ← API設計スキル
│   ├── SKILL.md                 ← スキル本体（必須ファイル）
│   └── references/              ← 詳細リファレンス
│       ├── openapi-patterns.md
│       └── rest-design-principles.md
│
├── performance-optimizer/       ← パフォーマンス最適化スキル
│   ├── SKILL.md
│   └── references/
│       ├── frontend-patterns.md
│       ├── database-patterns.md
│       └── bundle-analysis.md
│
├── a11y-audit/                  ← アクセシビリティ監査スキル
│   ├── SKILL.md
│   └── references/
│       ├── wcag-checklist.md
│       └── aria-patterns.md
│
└── dependency-audit/            ← 依存関係監査スキル
    ├── SKILL.md
    └── references/
        ├── license-compatibility.md
        └── update-strategies.md
```

**SKILL.md** = スキルの「脳」にあたるファイル。Claude Code はこれを読んで専門的に振る舞います。
**references/** = 詳細な参考資料。SKILL.md から必要なときだけ参照されます（トークン節約のため）。

## 用語集（非エンジニア向け）

| 用語 | 意味 |
|------|------|
| **API** | アプリ同士がデータをやりとりする窓口。「ユーザー情報ください」「はいどうぞ」のような通信の決まりごと |
| **REST** | API設計の代表的なルール。URLとHTTPメソッド（GET/POST等）で操作を表現する |
| **OpenAPI** | API設計書の世界標準フォーマット。YAMLというテキスト形式で書く |
| **パフォーマンス** | サイトの速さ・軽さ。「3秒以内に表示」等が目安 |
| **Core Web Vitals** | Googleが定めたサイト速度の3指標（LCP/INP/CLS） |
| **バンドルサイズ** | ブラウザに送られるJavaScriptファイルの大きさ。小さいほど速い |
| **アクセシビリティ (a11y)** | 障害のある方も含め、誰でもサイトを使えるようにすること |
| **WCAG** | Webアクセシビリティの国際基準。AA（ダブルエー）レベルが標準 |
| **ARIA** | HTMLだけでは伝えきれない情報をスクリーンリーダーに伝えるための属性 |
| **依存関係** | プロジェクトが使っている外部ライブラリ（部品）の集まり |
| **脆弱性** | セキュリティの穴。攻撃者に悪用される可能性がある弱点 |
| **ライセンス** | ソフトウェアの利用条件。MITは自由、GPLは条件付き |
| **npm** | JavaScriptのライブラリ管理ツール。package.jsonに使うライブラリを記録 |
